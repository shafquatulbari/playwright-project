# API Reference

Base URL: `http://localhost:4000`

## Authentication

All item endpoints require a Bearer token in the `Authorization` header:

```
Authorization: Bearer <token>
```

Tokens are obtained by registering or logging in. They are JWTs signed with the server's `JWT_SECRET` and expire after 7 days.

---

## Auth Endpoints

### POST /api/auth/register

Create a new user account and receive a JWT token.

**Request Body:**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | string | Yes | Display name shown in the UI header |
| `email` | string | Yes | Must be unique across all users |
| `password` | string | Yes | Hashed with bcrypt before storage |

**Success Response (200):**

```json
{
  "token": "eyJhbGciOiJIUzI1...",
  "user": {
    "id": "abc123",
    "email": "user@example.com",
    "name": "Test User"
  }
}
```

**Error Responses:**

| Status | Condition | Body |
| --- | --- | --- |
| 400 | Missing name, email, or password | `{ "error": "Name, email and password are required" }` |
| 409 | Email already registered | `{ "error": "Email already in use" }` |
| 500 | Server error | `{ "error": "<message>" }` |

**Example (curl):**

```bash
curl -X POST http://localhost:4000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name": "Test User", "email": "test@example.com", "password": "Password123!"}'
```

**Request Body:**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `email` | string | Yes | Registered email address |

**Success Response (200):**

```json
  "token": "eyJhbGciOiJIUzI1...",

```

**Error Responses:**

| Status | Condition | Body |
| --- | --- | --- |

**Example (curl):**

```bash
curl -X POST http://localhost:4000/api/auth/login \

```

## Item Endpoints (Protected)

All endpoints below require the `Authorization: Bearer <token>` header. Items are scoped to the authenticated user — you can only access your own items.

### Item Data Model

| Field | Type | Description |
| --- | --- | --- |
| `id` | string | Unique identifier (generated by NeDB) |
| `title` | string | The item's display title (required on creation) |
| `description` | string | Optional longer description |
| `column` | `"todo"` \| `"doing"` \| `"done"` | Which board column the item belongs to. Default: `"todo"` |
| `order` | number | Position within the column (1-indexed). Auto-assigned on creation. |
| `priority` | `"low"` \| `"normal"` \| `"high"` \| `"urgent"` | Item priority level. Default: `"normal"` |
| `createdAt` | number | Unix timestamp in milliseconds when the item was created |
| `updatedAt` | number | Unix timestamp in milliseconds of the last update |

---

### GET /api/items

Fetch all items for the authenticated user.

**Parameters:** None (user is determined from the JWT token)

**Response (200):** Array of Item objects, sorted by `column` ascending then `order` ascending.

```json
[
  {
    "id": "abc123",
    "title": "My Task",
    "description": "",
    "column": "todo",
    "order": 1,
    "priority": "normal",
    "createdAt": 1700000000000,
    "updatedAt": 1700000000000
  }
]
```

**Example (curl):**

```bash
curl http://localhost:4000/api/items \
  -H "Authorization: Bearer <token>"
```

---

### POST /api/items

Create a new item.

**Request Body:**

| Field | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `title` | string | Yes | — | Item title. Cannot be empty. |
| `description` | string | No | `""` | Optional description |
| `column` | string | No | `"todo"` | Initial column: `"todo"`, `"doing"`, or `"done"` |
| `priority` | string | No | `"normal"` | Priority: `"low"`, `"normal"`, `"high"`, or `"urgent"` |

**Success Response (201):** The created Item object with all fields populated including `id`, `createdAt`, `updatedAt`, and `order`.

**Error Responses:**

| Status | Condition | Body |
| --- | --- | --- |
| 400 | Missing or empty title | `{ "error": "Title is required" }` |
| 401 | Missing or invalid token | `{ "error": "Token required" }` |

**Example (curl):**

```bash
curl -X POST http://localhost:4000/api/items \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"title": "Buy groceries", "priority": "high"}'
```

---

### PUT /api/items/:id

Update specific fields of an existing item. Only the fields you include in the request body will be changed.

**URL Parameters:**

| Parameter | Description |
| --- | --- |
| `:id` | The item's unique ID |

**Request Body (all fields optional):**

| Field | Type | Description |
| --- | --- | --- |
| `title` | string | New title |
| `description` | string | New description |
| `column` | string | New column (`"todo"`, `"doing"`, `"done"`) |
| `priority` | string | New priority (`"low"`, `"normal"`, `"high"`, `"urgent"`) |

**Success Response (200):** The updated Item object. The `updatedAt` field is automatically set to the current timestamp.

**Error Responses:**

| Status | Condition | Body |
| --- | --- | --- |
| 401 | Missing or invalid token | `{ "error": "Token required" }` |
| 404 | Item not found or not owned by user | `{ "error": "Not found" }` |

**Example (curl):**

```bash
curl -X PUT http://localhost:4000/api/items/abc123 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"title": "Updated Title", "priority": "urgent"}'
```

---

### DELETE /api/items/:id

Delete an item permanently.

**URL Parameters:**

| Parameter | Description |
| --- | --- |
| `:id` | The item's unique ID |

**Success Response:** 204 No Content (empty body)

**Error Responses:**

| Status | Condition | Body |
| --- | --- | --- |
| 401 | Missing or invalid token | `{ "error": "Token required" }` |

**Example (curl):**

```bash
curl -X DELETE http://localhost:4000/api/items/abc123 \
  -H "Authorization: Bearer <token>"
```

### POST /api/items/reorder

Batch update the column and order of multiple items. Used by the frontend when items are dragged between columns or reordered within a column.

**Request Body:**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `orderMap` | object | Yes | Keys are column names (`"todo"`, `"doing"`, `"done"`). Values are arrays of item IDs in their new order. |

The position in the array determines the item's `order` value (index 0 = order 1, index 1 = order 2, etc.). Items are also moved to the column they appear under.

**Example Request:**

```json
{
  "orderMap": {
    "todo": ["id3"],
    "doing": ["id1", "id2"],
    "done": []
  }
}
```

This sets:
- `id3` → column "todo", order 1
- `id1` → column "doing", order 1
- `id2` → column "doing", order 2

**Success Response (200):** Array of all updated Item objects with new column, order, and updatedAt values.

**Example (curl):**

```bash
curl -X POST http://localhost:4000/api/items/reorder \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"orderMap": {"todo": [], "doing": ["id1", "id2"], "done": ["id3"]}}'
```

## Error Response Format

All errors return a JSON object with an `error` field:

```json
{ "error": "Description of what went wrong" }
```

Common status codes:
- **400** — Bad request (missing or invalid fields)
- **401** — Unauthorized (missing or invalid JWT token)
- **404** — Resource not found
- **409** — Conflict (e.g., duplicate email)
- **500** — Internal server error

---

## User Scoping

All item operations are scoped to the authenticated user via the JWT token:

- **GET /api/items** only returns items where `userId` matches the JWT claim
- **POST /api/items** automatically sets `userId` from the JWT
- **PUT /api/items/:id** only updates if the item's `userId` matches the JWT
- **DELETE /api/items/:id** only deletes if the item's `userId` matches the JWT
- **POST /api/items/reorder** only reorders items belonging to the authenticated user

This ensures complete data isolation between users.
